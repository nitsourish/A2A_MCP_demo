Weather Agent × Activity Planner Agent

Two toy agents, two protocols. A2A connects agent to agent over HTTP; MCP connects each agent to its own private tool server over stdio. The Activity Planner never touches a weather API directly — it asks the Weather Agent.

Request flow

  1. User sends "what should we do in Paris today?" to the Activity Planner Agent over A2A.
  2. Activity Planner extracts the city, then opens its own A2A call to the Weather Agent.
  3. Weather Agent calls get_current_weather on its MCP server over stdio.
  4. Weather Agent replies to the Activity Planner with a completed A2A Task.
  5. Activity Planner feeds the condition into its own recommend_activity MCP tool.
  6. Activity Planner returns a weather-aware suggestion to the user as a completed A2A Task.
Educational implementation — common/a2a_protocol.py mirrors the shape of the real A2A spec (agent cards, JSON-RPC message/send, task lifecycle) without depending on a2a-sdk. See projects/a2a-demo/README.md for setup and notebook/a2a_prototype.ipynb for a step-by-step walkthrough.
